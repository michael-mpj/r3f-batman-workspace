# Security Policy

## Supported Versions

Security updates are provided for the latest published version on the default branch.

| Version | Supported |
| ------- | --------- |
| Latest  | :white_check_mark: |
| Older   | :x: |

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub issues, discussions, or pull requests.

Instead, please report suspected vulnerabilities privately by using GitHub's **Private vulnerability reporting** feature for this repository, if available. If that option is not available, contact the maintainer through GitHub and request a private channel for disclosure.

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce the problem
- The affected version, commit, or workflow usage
- Any proof-of-concept, logs, or screenshots that help explain impact
- Your assessment of the potential severity and impact

You can expect an initial response within **5 business days**. After confirmation, we will work on a fix and coordinate disclosure as appropriate.

## Disclosure Policy

Please allow reasonable time for investigation and remediation before any public disclosure.

## Security Best Practices for Users

- Pin GitHub Actions to a full-length commit SHA where possible
- Review workflow permissions and use least privilege
- Keep dependencies and workflows up to date
